## 一、const 与指针的排列组合

两种基本限制：
- **指向的值能不能改？**
- **指针本身能不能换地址？**

const 修饰的是紧挨着它的东西，从右往左读：

### 1. pointer to const（指向常量的指针）

```cpp
const int* p = &x;
// 从右往左读：p 是一个 指针，指向 const int
```

| 操作 | 能否 |
|------|:----:|
| `*p = 10`（改值） | ❌ |
| `p = &y`（换地址） | ✅ |

> 用途：只读访问，不修改。常用于函数参数，保证不修改传入的数据。

### 2. const pointer（常量指针）

```cpp
int* const p = &x;
// 从右往左读：p 是一个 const 指针，指向 int
```

| 操作 | 能否 |
|------|:----:|
| `*p = 10`（改值） | ✅ |
| `p = &y`（换地址） | ❌ |

> 用途：固定指向某个变量，不允许换目标。

### 3. const pointer to const（常量指针指向常量）

```cpp
const int* const p = &x;
```

| 操作 | 能否 |
|------|:----:|
| `*p = 10`（改值） | ❌ |
| `p = &y`（换地址） | ❌ |

> 既不让你改值，也不让你换地址。极少用。

## 二、引用（Reference）

引用本质上是变量的**别名**，可以理解为**自带解引用的指针**。

### 声明与使用

```cpp
int x = 42;
int &r = x;   // r 是 x 的别名，& 在类型后面 = 声明引用

r = 100;      // 等价于 x = 100
cout << r;    // 等价于 cout << x
```

### 引用 vs 指针

| | 指针 | 引用 |
|---|---|---|
| 本质 | 存地址的变量 | 变量的别名 |
| 是否占用空间 | 是（存地址） | 底层也是地址，但对用户透明 |
| 能否为空 | ✅ 可以（NULL） | ❌ 必须初始化 |
| 能否换绑 | ✅ 可以 | ❌ 绑定后终身不变 |
| 使用方式 | `*p` 解引用 | 像普通变量一样直接用 |
| 函数参数 | `swap(&x, &y)` | `swap(x, y)` —— 更自然 |

### 引用不能换绑

```cpp
int x = 42, y = 99;
int &r = x;    // r 绑定 x

r = y;         // ❌ 不是换成 y 的别名！而是 x = y（x 变成 99）
```

一旦绑定，r 永远代表 x。

## 三、const 引用

```cpp
int x = 42;
const int &r = x;   // 不能通过 r 修改 x
// r = 100;         ❌ 编译错误
x = 100;            ✅ 直接改 x 完全可以
```

### 关键对比

```cpp
const int* p = &x;    // 不能通过 p 改 x
const int& r = x;     // 不能通过 r 改 x
// 两者效果一样：限制的是"通过这个途径改值"
```

### const 引用的典型用途：函数参数

```cpp
void print(const int &a) {   // 传引用，不复制
    cout << a;               // 只能读
    // a = 10;               ❌ 编译错误
}
```

两个好处：
1. **避免复制**（大对象时性能提升明显）
2. **保证安全**（函数不能修改原始数据）

## 四、`&` 符号的三种含义

```cpp
int x = 42;
int* p = &x;      // ① 取地址符 —— 写在表达式里，获取 x 的地址
int &r = x;       // ② 声明引用 —— 写在类型后面，& 是类型的一部分
int a = b & c;    // ③ 按位与运算符（以后学）
```

## 五、快速记忆表

| 写法 | 换指向？ | 改值？ | 名称 |
|------|:-------:|:-----:|------|
| `int* p` | ✅ | ✅ | 普通指针 |
| `const int* p` | ✅ | ❌ | pointer to const |
| `int* const p` | ❌ | ✅ | const pointer |
| `const int* const p` | ❌ | ❌ | const pointer to const |
| `int &r` | ❌ | ✅ | 引用 |
| `const int &r` | ❌ | ❌ | const 引用 |

**对比技巧：** const 在 `*` 左边 → 限制值；const 在 `*` 右边 → 限制指针本身。引用天生自带"右边 const"（不能换绑），所以只有加"左边 const"这一种玩法。
