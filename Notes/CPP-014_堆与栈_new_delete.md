# CPP-014：堆与栈、new/delete

**日期：** 2026-07-15  
**位置：** 阶段二 — 深入理解内存

---

## 一、栈（Stack）

**自动管理，用完就回收。**

```cpp
void func() {
    int x = 42;       // 栈上分配
    int arr[10];      // 栈上分配（大小编译期固定）
}                     // 离开作用域，自动销毁
```

| 特性 | 说明 |
|------|------|
| 速度 | 快（直接移动栈顶指针） |
| 大小 | 固定，通常几 MB |
| 生命周期 | 离开作用域自动销毁 |
| 分配时机 | 编译期确定 |

## 二、堆（Heap）

**手动管理，用完必须自己还。**

```cpp
int* p = new int(42);   // 堆上分配一个 int，值为 42
delete p;               // 手动释放

int* arr = new int[n];  // 堆上分配数组（大小运行期决定！）
delete[] arr;           // 释放数组必须加 []
```

| 特性 | 说明 |
|------|------|
| 速度 | 慢于栈（需要查找空闲内存） |
| 大小 | 大，可用 GB 级 |
| 生命周期 | 直到被 delete |
| 分配时机 | 运行期决定 |

## 三、比喻

```
栈 = 酒店大堂寄存柜     ← 自动管理，退房自动回收
堆 = 租仓库             ← 你租的你自己还，不还就占着
```

## 四、new 做了什么

```cpp
int* p = new int(42);
```

1. 在堆上分配一块能装下 int 的内存
2. 在这块内存中放入值 42
3. 返回这块内存的地址

```
图解：

栈：    p ────────→ 堆：┌─────┐
(存地址，指针本身)     │  42 │
                       └─────┘
```

`p` 本身在栈上（局部变量），`p` 指向的内存（`*p`）在堆上。

## 五、delete 做了什么

```cpp
delete p;
```

把堆上那块内存"还回去"。**delete 之后不能再访问 `*p`**——那块内存已经不属于你了。

## 六、配对规则 ⚠️

| 分配 | 释放 |
|------|------|
| `new` | `delete` |
| `new[]` | `delete[]` |

**混用是未定义行为**（可能崩溃）。

## 七、指针 vs 数组写法

```cpp
int* scores = new int[n];

scores[0] == *(scores + 0)    // 语法糖：[] 本质是指针偏移 + 解引用
scores[i] == *(scores + i)

// 指针加 1，地址移动 sizeof(int) = 4 字节
std::cout << scores;       // 首地址（假设 0x100）
std::cout << scores + 1;   // 0x104 （加 4 字节）
std::cout << scores + 2;   // 0x108
```

**核心认识：** `scores` 的类型是指针（`int*`），但 `scores[i]` 的用法像数组。在 C++ 中，"数组"很大程度上就是"指针 + 偏移量"。

## 八、项目对比：成绩统计

| 方案 | 内存 | 适用场景 |
|------|:----:|----------|
| 一边输入一边统计 | O(1) | 只要汇总数据（总分/平均分） |
| `new[]` 动态数组 | O(n) | 需要保存数据，后续排序/查找/重算 |

**没有绝对的好坏，根据需求选方案。**

## 九、内存泄漏

```cpp
for (int i = 0; i < 100000; i++) {
    int* p = new int(42);
    // ❌ 没有 delete —— 每次循环泄漏 4 字节
}
```

程序持续占用越来越多内存，最终耗尽可能崩溃或卡死。**程序退出后操作系统会自动回收**，但长期运行的服务不能依赖这个。

## 十、总结

| 分配位置 | 分配方式 | 释放方式 | 生命周期 |
|---------|---------|---------|---------|
| 栈 | `int x;` | 自动 | 离开作用域自动销毁 |
| 堆 | `int* p = new int;` | `delete p;` | 直到被 delete |
| 堆（数组）| `int* p = new int[n];` | `delete[] p;` | 直到被 delete |
