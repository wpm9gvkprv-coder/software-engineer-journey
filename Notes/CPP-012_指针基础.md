## 1. 为什么需要指针？

变量存在**内存**中，每个变量有一个**地址**（门牌号）。指针让我们能：
- 间接访问变量（不通过变量名，通过地址）
- 在函数之间传递地址，实现"真正修改外部变量"

## 2. 三个基本操作

| 操作 | 含义 | 类比 |
|------|------|------|
| `&x` | 取 x 的**地址** | 查 x 的门牌号 |
| `int* p` | 声明**指针变量** p | 准备一张写地址的纸条 |
| `*p` | **解引用** | 顺着纸条的地址找到房间里的值 |

```cpp
int x = 42;
int* p = &x;   // p 存 x 的地址

cout << p;     // 输出 x 的地址（门牌号）
cout << *p;    // 输出 42（房间里的东西）

*p = 100;      // 通过指针修改 x
               // 等价于 x = 100
```

## 3. 关键理解：一旦绑定，*p 就是变量本身

```cpp
int* p = &x;   
// 从此 *p 可以看作 x 的替身
// *p = 100 等价于 x = 100
```

## 4. 指针做函数参数（实现真正的交换）

### ❌ 值传递（失败）
```cpp
void swap(int a, int b) {   // a, b 是 x, y 的副本
    int temp = a;           // 交换的是副本，不影响 x, y
    a = b;
    b = temp;
}
```

**原因：** `a` 和 `x` 是不同房间，地址不同。交换 a, b 不影响 x, y。

### ✅ 指针传递（成功）
```cpp
void swap(int* p1, int* p2) {   // 接收地址
    int temp = *p1;              // *p1 就是 x 本身
    *p1 = *p2;                   // 直接修改 x
    *p2 = temp;                  // 直接修改 y
}

int main() {
    int x = 5, y = 10;
    int* p1 = &x;                // p1 存 x 的地址
    int* p2 = &y;                // p2 存 y 的地址
    swap(p1, p2);               // 传地址，不是传值
}
```

## 5. 易错点

**声明 vs 使用：**
```cpp
int* p;    // 声明：p 是指针类型
*p         // 使用：解引用，拿 p 指向的值
```

**传参时：**
```cpp
void func(int* p);  // 参数声明：p 是指针

func(p);            // ✅ 正确：p 本身是地址
func(*p);           // ❌ 错误：*p 是 int 值，类型不匹配
```

## 6. 野指针危险

```cpp
int* p3;     // 未初始化的指针 —— 指向随机地址
*p3 = 100;   // ❌ 往随机地址写值，可能崩溃
```

指针声明后必须指向有效的地址才能解引用。

## 7. 类比：指针 = 写地址的纸条

```
变量   = 一个有门牌号的房间
地址   = 门牌号
指针   = 写着门牌号的纸条
解引用 = 按纸条上门牌号找到房间
&      = 查房间的门牌号
*      = 按门牌号进房间
```
