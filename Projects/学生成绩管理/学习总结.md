## 项目需求

## 遇到的问题与解决过程

### 1. 编译器环境问题

**问题：** 从 `Code/` 目录切换到项目目录后，`cl.exe` 找不到源文件。  
**原因：** 当前工作目录是 `D:\ASoftwareEngineerJourney`，而 `main.cpp` 在 `Projects/学生成绩管理/` 子目录中。  
**解决：** 用 `cd Projects/学生成绩管理` 切换到项目目录再编译。

**收获：** 编译器的"找不到文件"错误不一定是编译器坏了，先检查**当前工作目录**。

### 2. VS Code IntelliSense 找不到头文件

**问题：** `#include <iostream>` 报错，VS Code 提示"请更新 includePath"。  
**原因：** VS Code 的 C++ 扩展不知道 MSVC 的头文件放在哪。  
**解决：** 创建 `.vscode/c_cpp_properties.json`，配置 `includePath` 指向 MSVC 和 Windows SDK 的头文件目录。

### 3. 核心 Bug：函数调用没加参数

```cpp
// ❌ 错误：isPass 是一个函数名，if 对它求值永远为 true
if(isPass){
    pass++;
}

// ✅ 正确：调用 isPass 函数并把 score 传进去
if(isPass(score)){
    pass++;
}
```

**教训：** 函数名本身是一个**指针**，它在条件表达式中永远为 true。**调用函数必须加括号和参数**。

### 4. 变量未初始化

```cpp
double total;        // ❌ 没有初始化，值是垃圾
double total = 0;    // ✅ 初始化为 0
```

**教训：** C++ 局部变量不会自动清零，使用前必须初始化。

### 5. 固定初始值的边界问题

**问题：** `lowest=100` 作为初始值，如果直接输入 -1 结束，最低分会显示 100。  
**最初写法：**
```cpp
int highest = 0;
int lowest = 100;
```

**改为初值法：** 第一个分数同时赋给 `highest` 和 `lowest`：
```cpp
if(person == 1){
    highest = score;
    lowest = score;
}
```

这样既解决了边界问题，也不需要用魔数（magic number）做初始值。

### 6. 代码优化：避免冗余判断

**第一次方案**将最高/最低分判断放在 `if(person==1)` 外面，导致第一个分数被重复比较：
- 先 `highest=score, lowest=score`
- 再判断 `highest<=score` → 总是 true，重新赋值（浪费）

**优化后：** 用 `else` 分支，第一次只赋值，后续只比较，逻辑更清晰。

## 最终代码结构

```
main()
├── 变量定义与初始化
├── while(1) 循环
│   ├── 输入分数
│   ├── if(score == -1) → break
│   ├── 累加总分、人数
│   ├── if(person == 1) → 设初值
│   │   else → 更新最高/最低分
│   └── if(isPass(score)) → 统计及格/不及格
├── 计算平均分（处理 person==0 边界）
└── 输出结果

isPass(score) → 判断 score >= 60
```

## 关键知识点总结

### 函数
- 声明与定义分离（先声明后使用）
- 参数传递：函数名(参数) 才是正确调用
- 返回值用 `return` 返回给调用方

### 变量
- **必须初始化**，否则值是未定义的（垃圾值）
- 类型选择：分数用 `int`，总分用 `double`，人数用 `int`

### 循环
- `while(1)` + `break` 实现"输入特定值结束"的模式
- `break` 跳出循环

### 条件判断
- `if-else` 分支结构
- 用 `person==1` 判断"是不是第一个"
- 处理边界情况（无数据输入）

### 调试思维
- 遇到 bug 先看症状 → 定位代码位置 → 分析原因
- 函数调用检查：有没有加括号？参数传对了没？
- 变量检查：初始化了没有？类型对不对？

## 对比：数组方案 vs 一边输入一边统计

| | 数组方案 | 本方案（B） |
|---|---|---|
| 存储 | 存所有分数 | 不存储 |
| 可以后续计算 | ✅ 可以重新计算 | ❌ 不能 |
| 内存 | O(n) | O(1) |
| 复杂度 | 更高（需要数组知识） | 更低 |

本方案用"遍历一次"的思路解决了问题，是**流式处理**思想的雏形——数据经过时处理完，不再保存。
